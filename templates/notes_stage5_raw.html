JavaScript (JS) is a programming language with its own syntax which adds more ways to create interactive web content. Like css, it may be added to the webpage in several ways:

  1. Write JS directly into the html file inside a set of <script> </script> tags
  2. Keep JS in a separate .js file

The second option is preferable as it keeps this this language code separate from our other languages such as html and css. Keeping all languages separate preserves better overview, correct syntax highlighting and eases debugging.

// CONCEPT END // // CONCEPT // JavaScript variables

Variables are declared a little differently in JS. They have to be prefixed by the keyword var:
var firstName = "Shelly";
var age = 32;

Apart from that, it works much the same way as Python; it takes just an equal sign to declare the value to the variable name. Strings are enclosed in quotes while numbers are not and there's no special requirement for differing between integers and floating point values. Other data types like arrays, functions and objects use the same var syntax.

// CONCEPT END // // CONCEPT // JavaScript debugging tip

JS can output to the browser's console (found in the browser Web Developer/Inspector tools). This is very useful to check for expected code behavior, see contents of variables as the code executes and in general for debugging JS code. This is very much similar to the trick where we used the Google App Engine (GAE) log to catch Python code output for debugging purposes.

// CONCEPT END // // CONCEPT // JavaScript Object Notation (datastructure)

JSON is a data interchange format that can convey some universal data structures between computers. It's also pretty straightforward for humans to read and write. The JSON data structures are called: object, array, value, string and number.

There is nothing new in terms of those structures when comparing to what has been learned during this course already and comparing to Python brings back resemblance for the structures, e.g.:

  • JSON array ~ this resembles a Python list (an ordered collection of values)
  • JSON object ~ this resembles a Python dictionary or object (an unordered set of name/value pairs)
// CONCEPT END // // LESSON // Lesson 5.2: The power of APIs // CONCEPT // XML (datastructure)

eXtensible Markup Language or XML is another data interchange format and different from JSON. It is structured to resemble the HTML DOM more and follow W3C and several other related specifications all of which are free open standards. XML does not allow void tags, i.e. all tags must close with a closing tag unlike current HTML formats.

An API or Application Programming Interface – is a direct and standardized interface used to communicate with servers. Instead of harvesting results from a webpage's display and getting hard to interpret html back, the API returns a much more useful data format - e.g. in the xml or JSON format.

// CONCEPT END // // CONCEPT // Mini-DOM

This can be imported in Python from the library dom: from xml.dom import minidom. Strings of xml can be parsed to a DOM structure using p = minidom.parseString(xmlstring). The minidom stored in this p object can hereafter be manipulated further and represented via other functions in the object. A print dir(p) will reveal how many object methods are available within the mini-DOM. Examples are some look-up functions: getElementsById, getElementsByTagName, getElementsByTagNameNS. Useful also are parentNode, childNodes, nodeValue and toprettyxml. // CONCEPT END // // CONCEPT // JSON

Like minidom, json (the JavaScript Object Notation) can also be imported into python: import json to hold data strings of structures adhering to this particular notation. Any string holding a json data structure can now be loaded into a python variable: p = json.loads(string) and from there be manipulated as any other typical Python data structure list or dictionary data.

// CONCEPT END // // CONCEPT // Internet citizenship

Being a good internet citizen:

  • Use a good user-agent – specify it with urllib
  • Rate-limit yourself – limit the speed with which you send requests to servers, e.g. by adding a pause with the sleep function from the time library
  • Use common protocols and data formats – use xml or json instead of inventing your own, enables others to easily communicate with and build on your creations

This exercise uses the API of hostip.com to obtain GPS longitude and latitude coordinates for any given ip address. These coordinates are then coupled with Google Maps Image APIs for Static Maps to draw a map where we can place markers of the ips we look up on hostip.com. It was a useful insight into talking with APIs.

Hostip.com has a clear guidance for using their API on their web page. You simply send a GET to a special address including the ip to look up and it returns a response in either JSON, XML or HTML. An example of a GET could be:
http://api.hostip.info/get_json.php?ip=12.215.42.19&position=true
which would spit out the response in JSON format:
{"country_name":"UNITED STATES","country_code":"US","city":"Aurora, TX","ip":"12.215.42.19","lat":"33.0582","lng":"-97.5159"}

Knowing the syntax of JSON it is now an easy piece to load up this data structure module in your preferred programming language (say Python) and pluck out the desired GPS coordinates from the response.

// CONCEPT END // // LESSON // Lesson 5.3: Recursion and Parallel Computing // CONCEPT // Recursion

Recursion is a circular process where an initial element is processed and included into the product of this process. For a recursive process to be able to end, it needs to have a rule for stopping again. So recursion requires at least 2 rules, one for recursion (recursive case) and one termination/ending rule (base case).

Programatically, this is can be implemented by writing a function that calls itself with a slight variation (recursive case) but also checks for a terminating condition (end case), e.g.: 

  def factorial(n):
    # Base case, f(0)=1
    if n == 0:
      return 1
    else:
      # Recursive case
      return n * factorial(n - 1)

  def is_palindrome(s):
    # Base case, if empty string is left, it indicates palindrome found
    if s == '':
      return True
    elif s[0] == s[-1]:
      # Recursive case, if first and last character match, call again
      return is_palindrome(s[1:-1])
    else:
      return False
// CONCEPT END // // CONCEPT // Parallel computing

Larger amounts of data sets are broken into lesser chunks of data sets and processed in parallel. The data has to be of a structure so that it can be chopped into smaller sets and still maintain a regular structure (e.g. sales numbers from vendors in various regions).

When split up, the processing is done by programs doing the exact same thing in parallel to each of its own data part. When all the programs running in parallel have processed their data, the outputs are passed to a second function or functions. This second processing compiles the inputs after certain rules (e.g. adds intermediate sums for same regions).

// CONCEPT END // // CONCEPT // Dash project 1 - style techniques

A quick nice page can be displayed and made flexible on various display formats with some simple tricks like centering everything and adding an auto-resizing background. This can be set in css: 

  body {
      text-align: center;
      background: url("http://www.whereever.com/somepic.png");
      background-size: cover;
      background-position: center;
  }

A limited amount of elements on the page and a nice big readable font also go a long way.

// CONCEPT END // // CONCEPT // Dash project 2 - media queries, scripting

Another trick is to wrap all text blocks into a block tag set like the <article></article> tags and set the css property for that tag to margin: 0 auto; which sets top+bottom margins to 0 but ensures left+right margins are auto-adjusted effectlively keeping all article text blocks to the center of the display. Using the max-width property also ensures display adapts to less-than browser widths: 

  article {
      max-width: 500px;
      padding: 20px;
      margin: 0 auto;
  }

It's possible to test for conditions and activate alternative css if the recommended settings encoded in the css styling is compromised, e.g. by browser resizing or viewing on incompatible devices. In this example, css is asking the media if display width drops below 500 pixels. In that case, the css of the body is changed to a red background (maybe serving as a warning to the user): 

  @media (max-width: 500px) {
      body {
          background: red;
      }
  }

Obvious alternatives in the above would be to implement css that adapts for all elements as the display width narrows, e.g. smaller h1 font size, less padding, smaller border, inlined list items changed to display as block etc.

// CONCEPT END // // CONCEPT // Dash project 3 – gradients, mores scripts, more fonts, more, more, more

Setting max-width property of block elements to 600 pixels is currently a good mobile proof setting since many phones have 600+ pixels width in landscape mode and most tablets likewise regardless of orientation.

There are cool new features on the way in CSS3 but for now they have to be called specifically with a "vendor-prefix" like the gradient feature here that only some browsers are currently able to produce: 

  background: -webkit-linear-gradient(bottom, rgba(0,0,0,1), rgba(0,0,0,.4));
  background: -moz-linear-gradient(bottom, rgba(0,0,0,1), rgba(0,0,0,.4));

When using on the edge features or not globally implemented features one should always set the current best alternative on a line before so that will execute for older or non-compatibel browsers. For the above example with gradient, one would simply add a a line before setting background to a uniform color that all browsers can interpret.
